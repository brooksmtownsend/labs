Manage User Access

  1. From the AWS Management Console -> Find Services search for IAM (Manage User Access and Encryption Key).

  2. Click Policies from the Navigation.

  3. It is recommended that you create a new limited access policy rather than giving a user admin access. For the purpose of the Critical Stack install we will create a new policy with only the permissions needed to build the cluster. Click Create policy .

  4. In the new Create policy window you can create a new policy with the Visual editor or with JSON (JavaScript Object Notation). Select JSON

  6. Your policy should look something like this:

  7. Select Review Policy.

  8. Give your policy a name and description and select Create policy.

  9. Click Users from the Navigation and Add user.

  10. Select Programmatic access for the Access Type and Click on Next: Permission at the bottom of the page.

  11. Create a new group for permissions by selecting Create group.

  12. Click Refresh and search for the policy you just created CS_Install_Policy. Click the checkbox by the policy name and give your group a name. Select Create Group.

  13. With group selected click Next: Tags from the Add user screen.

  14. Add any optional tags and select Next: Review.

  15. Select Create user to finalize to user creation.

  16. The Access key ID and Secret access key will be used for the CS Installer.
